# Architecture Decision Records (ADR)

코어 기술 결정의 **배경 / 후보 비교 / 결정 / 재검토 조건**을 박제하는 디렉토리.

## 규약

- 파일명: `<YYYYMMDD>-<kebab-case-topic>.md`
- **같은 날짜에 여러 ADR 이 발생할 경우 topic 접미사로 관계 표현**: 상위 결정은 `<topic>` (예: `-gitflow-main-develop`), 세부 결정은 `<topic>-<하위범위>` (예: `-release-merge-strategy`). 상위 ADR 은 하위 ADR 에서 "선행 ADR" 로 인용
- 본문 구조와 작성 절차는 `record-adr` 스킬을 참조 (또는 호출).
- 결정이 무효화/대체될 때는 **삭제하지 않고** 상태를 `Superseded by [신규 ADR]`로 갱신한다.
- 1 ADR = 1 결정. 여러 결정을 묶지 않는다.

## §Amendments — 갱신 이력 박제 규약 (#226-3)

ADR 은 원칙적으로 **결정 시점 스냅샷** 이지만, 실측/환경 변화로 "재검토 조건" 이 트리거되어
수치·임계·옵션을 갱신해야 할 때가 있다. 본문을 직접 덮어쓰면 갱신 이전 근거와 변경 사유가
소실되므로, **파일 말미에 `## §Amendments` 섹션을 두고 테이블로 이력을 쌓는다**.

### 표준 포맷

```markdown
## §Amendments

| 날짜       | 변경 요약                                    | 근거 PR/이슈       |
| ---------- | -------------------------------------------- | ------------------ |
| 2026-04-18 | CI 임계 `> 7분 → > 11분` (verify-and-rust)   | #210 / PR #222     |
| 2026-04-18 | 금성 DoD ±1.0% → ±1.5% (10c 잔차 1.39%)      | #206 Phase C 진단  |

### 상세 (선택)

날짜 앵커(`### 2026-04-18`)로 세부 근거·측정값·영향 범위를 별도 기술.
테이블이 self-explanatory 면 상세 섹션은 생략 가능.
```

### 작성 정책

- **본문 숫자 직접 수정 금지** — 대신 본문은 원래 수치 유지 + `§Amendments` 테이블에 차이 박제.
  (예외: 단순 오타/링크 수정은 본문 직접 수정 후 `§Amendments` 생략 가능)
- **갱신 이력 테이블 컬럼 고정**: 날짜(YYYY-MM-DD) / 변경 요약(1줄) / 근거 PR 또는 이슈 번호
- **상세 근거는 테이블 아래 `### <날짜>` 앵커로 확장** — 이력이 누적돼도 테이블은 한눈에 읽힌다
- **재검토 조건 트리거 식별**: `§재검토 조건` 섹션의 어느 항목이 발동됐는지 변경 요약에 명시
  (예: "재검토 조건 3 발동 — CI 11분 임계 실측 도달")

### 소급 적용 정책

- **기존 ADR 에 강제 소급 적용 안 함** — 갱신 필요가 없는 ADR 을 수정하려 새 amendment 를
  억지로 발굴하지 말 것
- **신규 ADR 은 §Amendments 섹션 자리만 비워두고 생성** (템플릿은 `record-adr` 스킬 관리).
  첫 amendment 발생 시 실제 테이블 행을 추가
- 본 규약 도입 이전 ADR(2026-04-19 이전) 에 이미 갱신이 일어났다면, 다음 갱신 시점에 기존 수정을
  소급 박제해도 무방 (회고적 정리 허용)

### 시범 적용

`20260418-p7-integrator-upgrade.md` 에 `§Amendments` 섹션을 신설하고 3행을 박제:

- **2026-04-18 Phase C 진단** — "EIH structural bias" 가설 기각, 지구 DoD 1c→3c 확대, 금성
  ±1.0%→±1.5% (근거 #206 / PR #212) — 본 규약 도입 시점에 회고적 정리 허용 케이스
- **2026-04-18 CI 임계 갱신** — `verify-and-rust` 재검토 임계 `> 7분 → > 11분` (근거 #210 /
  #215 / PR #222) — 실측 9m15s + 20% 마진 반영
- **2026-04-19 §Amendments 섹션 신설** — 위 2건 소급 박제 + 본 규약 도입 자기참조 (근거 #226 /
  본 README.md)

## 언제 작성하는가

- 코어 언어/런타임/프레임워크 도입·교체
- 주요 외부 의존성 추가 (DB, 메시지 큐, 인증 등)
- 프로젝트 전반 영향 패턴 채택 (상태 관리, 빌드 도구, 모노레포 구조)

일회성 코드 결정은 ADR 대상이 아니다 (PR 본문/커밋 메시지로 충분).
